Azure Lab Architecture

Terraform Base Lab + SOC Layer — Project Roadmap

Project structure — click each phase

Terraform Azure Base Lab IaC Foundation

The reusable core. Every future project runs terraform apply -var-file=soc.tfvars (or whatever context) and gets a clean, consistent environment spun up in minutes. This is what makes the repo a portfolio piece and not just a one-off script.

Networking module

VNet, subnets (mgmt, workload, monitoring), NSGs with parameterized rules. Reusable across all projects.

Compute module

Variable VM count, size, OS. Toggle Windows vs Linux. Used to spin up attack/victim machines in SOC phase.

Log Analytics workspace

Deployed by default. Every VM ships logs here automatically. SOC phase builds on top of this.

Remote state

State stored in Azure Blob Storage — shows production habits, not just local dev.

tfvars profiles

soc.tfvars, hardening.tfvars, etc. Swap context without touching core modules. This is the "adaptable" piece.

PowerShell post-config

Runs after Terraform — installs agents, applies baseline GPO, joins domain. Bridges your existing skill set.

Skills demonstrated

Terraform Azure networking NSGs IaC modules Remote state PowerShell
SOC Layer — Sentinel + Detection Security

Deploy on top of Phase 1 using the SOC tfvars profile. The Log Analytics workspace is already running — you're just adding Sentinel, data connectors, and detection logic. This is where the project gets resume-worthy for security roles.

Microsoft Sentinel

Enable on the existing Log Analytics workspace. Connect data sources: Azure AD, VMs, Defender, Office 365 activity.

Attack simulation

Simulate brute force (failed logins), lateral movement, port scans from a separate "attacker" VM. Generate real alerts.

KQL detection rules

Write 3–5 custom analytics rules. Failed login threshold, impossible travel, suspicious process spawn. Document each one.

Automated playbooks

Logic App triggered by an alert — auto-disable compromised account, send email notification, create incident ticket.

Workbooks / dashboards

Build a Sentinel workbook showing alert trends, top offenders, geo map of logins. Screenshot this for your resume/LinkedIn.

Defender integration

Connect Microsoft Defender for Endpoint — shows you can unify EDR and SIEM, which is exactly what SC-200 tests.

Skills demonstrated

Microsoft Sentinel KQL SIEM Logic Apps Threat detection Defender integration
Future expansions Optional

Because the base lab is modular, these are just new tfvars profiles + modules dropped in. Each one adds a resume bullet and potentially a new cert path.

Intune hardening module

Enroll the lab VMs into Intune, apply your compliance policies. Publishes your existing work as documented, reproducible config.

AWS module

Add a Terraform AWS provider — spin up EC2, VPC, S3. Cross-cloud lab. Moves AWS from "basic exposure" to demonstrated.

Docker workloads

Deploy a containerized app (nginx, a Python script) on a lab VM. Not deep DevOps, but enough to add Docker legitimately.

Python cost monitor

The Azure/AWS cost monitoring script from the earlier roadmap. Plug it into this lab as the environment it monitors.

Cert alignment

SC-200 → Phase 2 Terraform Associate → Phase 1 AWS SAA → Phase 3 AWS module
Suggested GitHub repo structure Repository

Keep modules generic and profiles context-specific. Anyone cloning this should be able to spin up their own lab by editing one tfvars file.
